# Fairness Report - LendingClub

## Executive summary

This report summarizes the fairness evidence already present in `Fairness.md`.
It does not recompute metrics. The original workflow trained loan outcome models
and used AIF360 to measure group differences across a ZIP3-derived demographic
proxy.

The most important result: reweighing reduced the reported training mean outcome
difference from `-0.017565` to `0.000000`, and reduced the reported test mean
outcome difference from `-0.019698` to `-0.002036`.

## Prediction task

- Positive label: `Fully Paid`.
- Negative label: `Charged Off`.
- Protected-proxy attribute: `underprivileged`, derived from ZIP3-linked Census
  racial composition.
- Fairness toolkit: AIF360.

## Final model note

The original fairness workflow fits a random forest with instance weights from
the transformed training dataset. The rendered notebook reports:

| Metric | Value |
| --- | ---: |
| Random forest precision | 0.913286 |
| Random forest test score | 0.506695 |

The project discussion emphasizes high precision and ranked investor selection.
The reported `score` should not be interpreted as the only success criterion.

## Key risks

- ZIP3 proxy groups can expose aggregate disparity, but they are not individual
  protected-class labels.
- Reweighing reduces one measured disparity but does not guarantee fairness
  under every threshold, metric, or deployment setting.
- The raw data and runtime are not yet packaged for one-command reproduction.
